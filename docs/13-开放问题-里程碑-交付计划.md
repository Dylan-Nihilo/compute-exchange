## 13.2 需求优先级（MoSCoW，按模块）

### Must（v1 必须，缺则不可上线）
- 统一账户 + 全角色注册登录 + KYC + 资质审核（docs/06）
- 机房入驻审核 + 资质台账（排幽灵算力）（docs/03、09）
- 算力商品挂牌 + 检索 + 下单 + 余量锁定 + 现货撮合（docs/03）
- 易宝进件 + 合单支付 + 分账 + 退款 + 对账（docs/04）
- Token 工厂：独立 newapi 对接 + 套餐购买 + 额度同步 + API Key + 用量账单（docs/05）
- 多端工作台（买家/供给方核心，厂商基础）（docs/06）
- 运营后台：审核/商品/订单/资金/CRM/权限/合规开关（docs/08）
- 风控合规：合规承诺 + 挖矿排除/冻结 + 审计日志（docs/09）
- 居间需求登记 + CRM + 佣金台账（docs/07）
- 融资租赁展示 + 留资 + 合规声明（docs/07）
- 安全/等保三级技术项 + 备份（docs/10）
- 电子合同、发票（基础）（docs/03）

### Should（v1 应做）
- 自动调价、限时抢购、阶梯定价
- 信用分、SLA 记录、工单售后
- 资方工作台、施工项目进度管理、进销存台账
- 数据看板（运营/各端）、消息中心
- SEO、埋点

### Could（v1 可做可不做）
- 需求挂单双边撮合（求购单）
- 第三方登录、子账户成员
- 自动续费、收藏比价高级功能

### Won't（v1 明确不做）
- 底层算力调度/隔离网关（第三层技术中台）
- 融资租赁系统化撮合/放款
- 算力期货/份额/理财类（合规红线，永不做）
- App/小程序原生端、多语言、跨境结算

## 13.3 里程碑（开发节奏，待 Q-22 上线时间确认后细化）

| 里程碑 | 范围 | 依赖 |
|--------|------|------|
| M0 架构设计 | 技术选型、系统拆分、数据模型、接口设计、部署方案、SBOM | 本需求文档评审通过 |
| M1 账户与门户 | 账户/KYC/权限、门户官网、CMS、五大板块展示、留资CRM | M0 |
| M2 交易核心 | 机房入驻审核、算力挂牌、检索下单、现货撮合、订单履约 | M1 |
| M3 支付分账 | 易宝进件、合单支付、分账、退款、对账（先沙箱） | M2、易宝商务开通 |
| M4 Token 工厂 | 独立 newapi 部署、套餐、额度同步、API Key、账单 | M1、newapi 选型 |
| M5 风控与后台完善 | 挖矿排除/冻结、审计、运营后台全功能、合规开关 | M2-M4 |
| M6 合规联调与灰度上线 | 等保整改、备案、EDI/支付齐备、灰度→全量 | M3-M5 + 证照 |

> 开发可与合规并行：M1-M5 不依赖证照即可开发联调（支付走沙箱）；M6 真实交易开闸依赖证照（第11章时间线）。

## 13.4 验收口径（v1 整体）
1. **交易闭环**：机房入驻→审核→进件→挂牌→买家下单→合单支付→分账→对账→发票，沙箱/真实至少一条走通。
2. **Token 闭环**：购买套餐→支付→newapi 充值→API Key→调用成功→用量账单。
3. **多端**：4 类角色自助注册→审核→进入对应工作台，权限正确。
4. **居间/融资**：线索登记→CRM→分配→台账；融资留资含合规声明。
5. **合规**：挖矿排除/冻结闭环、零二清、备案号展示、审计日志、合规开关灰度可控。
6. **NFR**：等保三级技术项、HTTPS、密钥管理、限流防护、备份达标。

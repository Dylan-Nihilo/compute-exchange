# 04 · 功能需求 · 支付结算与分账（易宝支付）

> 需求编号：`REQ-B-xxx`。
> 核心红线：**资金严禁进入平台自有账户做二次清算（二清）**。平台以易宝**服务商**身份接入，机房作为**二级商户**进件，通过**合单支付 + 分账**实现"买家一笔付款 → 机房应得款进机房账户 + 平台佣金进平台账户"。详见第11章。

## 4.1 接入模式（基于易宝官方能力调研）

| 项 | 方案 |
|----|------|
| 平台角色 | 易宝**服务商（SAAS 服务商）**，由易宝商务侧开通，获取 SAAS 服务商编号、应用标识 |
| 机房/厂商角色 | **二级商户（特约商户）进件**，企业/个体类型，走特约商户入网接口或自助扫码入网 |
| 收款方式 | **聚合支付统一下单**（微信/支付宝/银行卡） |
| 多方收款 | **合单支付**：一笔订单子单分别入各收款商户账户 |
| 平台抽佣 | 叠加**分账**：从交易资金中按规则收取平台佣金 |
| 退款 | 申请退款 + 申请分账资金归还 |
| 密钥 | 国际 RSA2048，商户后台开发者中心配置公钥，私钥本地保管 |

> ⚠️ 实施前置（须商务沟通）：需联系易宝开通聚合支付、合单支付、分账、结算等产品能力，并配置分账规则与通知地址。默认结算策略可能导致二级商户结算账户缺少可退款资金——对接时必须书面确认结算方式、延迟分账时长和退款资金留存策略，并经沙箱验证。

## 4.2 商户进件（入网）

- **REQ-B-001 (P0)** 机房/设备厂商资质审核通过后，触发易宝二级商户进件流程：采集工商信息（名称、统一社会信用代码、注册地址、经营范围、证照有效期）、法人信息、联系人信息、结算银行账户。
- **REQ-B-002 (P0)** 进件资料上传与校验：营业执照、开户许可证/银行卡、法人身份证正反面、行业资质（与经营范围一致性校验）。
- **REQ-B-003 (P0)** 进件状态回填与查询：调用易宝入网进度查询接口，回显"进件中/成功/驳回(原因)"，驳回可补件重提。
- **REQ-B-004 (P1)** 费率配置查询/变更：展示该二级商户的支付产品费率。
- **REQ-B-005 (P1)** 经营范围变更管控：商户营业执照/经营范围变更需提前通知并重新审核（易宝合规要求），系统提供变更申报入口。

## 4.3 订单直付与支付

> 已决策：v1 不提供现金余额或预充值，买家下单后直接支付。Token 额度属于已购服务配额，不是资金余额。

- **REQ-B-011 (P0)** 订单直付：买家下单后用聚合支付统一下单直接付款（微信/支付宝/银行卡）。
- **REQ-B-012 (P0)** **合单支付**：购物车多商品/多供给方场景，一笔支付，子单分别入各机房账户；子单含分账标识。
- **REQ-B-013 (P0)** 支付结果异步通知与主动查询双保险：回调验签 + 定时对账补单，保证最终一致。
- **REQ-B-014 (P0)** 支付超时关单：未支付订单超时关闭并释放算力余量锁定（与 REQ-A-023 联动）。

## 4.4 分账

- **REQ-B-020 (P0)** 分账规则引擎：按订单计算"机房应得 + 平台佣金（撮合费率）+ 通道费"，生成分账指令。
- **REQ-B-021 (P0)** 触发式分账：支付成功后只生成`延迟中`分账记录；买家确认交付或无争议超时后，才按规则申请并完结分账。记录全部流水与结果（成功/失败/重试）。
- **REQ-B-022 (P0)** 分账失败重试与告警；可人工复核（运营后台）。
- **REQ-B-023 (P0)** 分账时间策略：支持延迟分账；买家确认交付或超时自动确认后再向机房分账，作为履约保证。具体资金留存能力由易宝产品确认，见已决策问题 Q-03。
- **REQ-B-024 (P1)** 佣金/分账规则可按品类、供给方、活动差异化配置。

## 4.5 退款与资金归还

- **REQ-B-030 (P0)** 退款申请：买家发起或运营发起，校验订单状态、履约情况、是否已分账。
- **REQ-B-031 (P0)** 已分账订单退款：调用"申请分账资金归还"先归还机房侧资金，再原路退款给买家；更新三方账目。
- **REQ-B-032 (P1)** 部分退款（按未消费时长/比例）。
- **REQ-B-033 (P0)** 风控冻结订单的资金处置：冻结期间暂停分账，结论后决定退款或放行（与 docs/09 联动）。

## 4.6 对账与财务

- **REQ-B-040 (P0)** 平台与易宝对账：每日拉取交易/分账/退款流水，与平台订单逐笔勾对，生成差异报表。
- **REQ-B-041 (P0)** 三方账目台账：每笔订单的买家付款、机房应得、平台佣金、通道费、退款、净额可追溯。
- **REQ-B-042 (P1)** 供给方结算只读视图：展示易宝账户中的待结算/可提现信息；平台不发起提现，仅提供进入易宝侧结算入口。
- **REQ-B-043 (P1)** 财务导出：对账单、分账明细、佣金报表导出（Excel/CSV）。

## 4.7 安全与合规

- **REQ-B-050 (P0)** 所有支付/分账接口调用验签（RSA2048），密钥安全存储（KMS/密钥管理），私钥不入库不入前端。
- **REQ-B-051 (P0)** 支付相关操作全留痕审计（操作人、时间、金额、结果）。
- **REQ-B-052 (P0)** **二清自查**：架构上保证资金不经平台自有账户中转；如运营/法务确认采用"备付金/担保支付"等模式，必须以易宝持牌账户实现，平台仅发指令。
- **REQ-B-053 (P1)** 资金异常监控：大额、高频、与经营范围不符的交易告警（易宝侧也会风控，平台侧做二次监控）。

## 4.8 待对接确认（移交架构阶段）
- 易宝具体产品组合（聚合/合单/分账/结算）与默认结算策略 → 商务确认。
- 易宝是否支持所需的延迟分账与退款资金留存策略 → Q-03，商务确认产品能力。
- 个体/小微 vs 企业二级商户类型覆盖范围。

## 4.9 验收要点
1. 沙箱环境完成：进件成功 → 合单支付成功 → 分账成功（机房+平台佣金）→ 退款+资金归还成功 → 对账平。
2. 全程无任何资金经过平台自有银行账户。
3. 回调验签、超时关单、失败重试、对账补单均可用。

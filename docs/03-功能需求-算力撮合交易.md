# 03 · 功能需求 · 算力撮合交易（交易所核心）

> 需求编号规则：`REQ-A-xxx`。优先级 P0=v1必做 / P1=v1应做 / P2=可后置。
> 本模块对标"全国一体化算力网撮合标准"的精神做轻量落地，但 v1 不接入官方算力网（见开放问题 Q-10）。

## 3.1 供给端：机房入驻与算力上架

### 3.1.1 机房入驻（资质强审核）
- **REQ-A-001 (P0)** 机房供给方提交入驻申请，填写：主体名称、统一社会信用代码、机房地址、所在省市、联系人、电信/IDC 相关资质证照（上传）、电力与散热说明、是否持证 IDC。
- **REQ-A-002 (P0)** 系统建立**机房资质审核台账**：每条记录含资质类型、证照号、有效期、审核人、审核时间、状态、附件。用于排除"幽灵算力"。
- **REQ-A-003 (P0)** 运营审核：通过 / 驳回（带原因）/ 要求补件。通过后该供给方解锁"上架"权限并触发易宝二级商户进件（docs/04）。
- **REQ-A-004 (P1)** 资质到期前 30 天自动预警，过期自动冻结其在售商品。
- **REQ-A-005 (P1)** 黑名单/灰名单：命中（涉挖矿、被处罚、虚假资质）禁止入驻。

### 3.1.2 算力商品上架
- **REQ-A-010 (P0)** 供给方发布算力商品，字段：
  - GPU 型号（如 H100/H800/A100/4090 等，枚举+自定义）、卡数、单机规格（CPU/内存/显存/存储/带宽）。
  - 交付形态：裸金属 / 容器 / 虚机 / 整机柜。
  - 计费模式：分时（元/卡·小时）、包月（元/卡·月）、包周等。
  - **可售时段**（针对余量算力：填闲置时间段，如 22:00–08:00）。
  - 余量（可售卡数/台数）、起订量、最短租期。
  - 地域、网络出口、是否支持公网、是否独享。
  - 合规承诺勾选（不得用于虚拟货币挖矿等，见 docs/09）。
- **REQ-A-011 (P0)** 商品上架前经运营/规则审核（敏感字段、价格异常、合规承诺）。
- **REQ-A-012 (P0)** 余量实时更新：成交/释放自动增减可售量；售罄自动下架。
- **REQ-A-013 (P1)** **自动调价**：供给方可设规则（按时段、按余量水位、按竞品价）自动浮动单价；平台留人工干预开关。
- **REQ-A-014 (P1)** 平台自营商品（包销/集采余量）：运营以"平台供给方"身份上架，差价归平台。
- **REQ-A-015 (P2)** 阶梯定价：按租期/规模设阶梯单价。

## 3.2 需求端：检索、选型、下单

- **REQ-A-020 (P0)** 算力商品列表与详情：支持按 GPU 型号、地域、计费模式、价格区间、可用时段、卡数筛选与排序（价格/余量/上新）。
- **REQ-A-021 (P0)** 比价视图：同型号不同供给方横向对比（单价、规格、地域、信用）。
- **REQ-A-022 (P0)** 下单：选择规格、数量、租期/时段、计费模式，实时试算总价（含税、含平台费说明）。
- **REQ-A-023 (P0)** 库存锁定：下单进入待支付时锁定余量，超时（如 15 分钟）未支付自动释放。
- **REQ-A-024 (P1)** 购物车 / 合单：一次下单多个商品（对应易宝合单支付，docs/04）。
- **REQ-A-025 (P1)** 续费：到期前/后在原订单上续租。
- **REQ-A-026 (P2)** 询价/议价：大额需求提交询价单，供给方/运营线下报价后转订单。

## 3.3 撮合与定价机制

- **REQ-A-030 (P0)** **现货撮合**：买家下单即时匹配在售商品并锁定（v1 以"挂牌-下单即成交"的现货撮合为主，非连续竞价撮合）。
- **REQ-A-031 (P1)** **限时抢购/余量秒杀**：对低价余量算力设限时、限量抢购活动。
- **REQ-A-032 (P1)** 阶梯定价规则引擎（与 REQ-A-015 配合）。
- **REQ-A-033 (P2)** 需求挂单撮合：买家也可挂"求购单"（目标价/规格），系统匹配供给（双边撮合）——v1 可后置。
- **REQ-A-034 (P0)** 平台费率：可按品类、供给方、活动配置撮合佣金率（3%–8% 区间，默认值可配），下单时计入分账规则。

## 3.4 订单履约与监管

- **REQ-A-040 (P0)** 订单生命周期管理（状态机见 02 §2.6），关键节点通知（站内信/短信/邮件可配）。
- **REQ-A-041 (P0)** 支付成功后进入"待开通"，供给方接单并标记"开通中→已交付"，回填交付凭证（实例 IP/账号/控制台链接/凭据，敏感信息加密存储）。
  - ⚠️ v1 **不做底层自动编排**：开通可由供给方在自有控制台操作后回填，或运营协助。见开放问题 Q-09。
- **REQ-A-042 (P0)** 履约监管：订单运行期内的状态、到期提醒、异常上报；接入风控（挖矿识别，docs/09）。
- **REQ-A-043 (P0)** 到期处理：到期前提醒续费；到期自动置"已完成"并释放余量。
- **REQ-A-044 (P1)** 工单/售后：买家可对订单发起工单（故障、不可用、申诉），运营介入。
- **REQ-A-045 (P1)** 履约 SLA 记录：可用率/故障时长记录，作为退款或信用依据。

## 3.5 结算、对账、合同、发票

- **REQ-A-050 (P0)** 平台统一对账：按订单生成"机房应得 / 平台佣金 / 支付通道费"三方账目，对接分账结果（docs/04）。
- **REQ-A-051 (P0)** 供给方结算单：周期（日/周/月可配）汇总应结金额、已分账金额、待结金额，可导出。
- **REQ-A-052 (P0)** **电子合同**：下单成交生成平台-买家、平台-供给方电子协议（模板化），存证（见开放问题 Q-11 电子签渠道）。
- **REQ-A-053 (P0)** **发票管理**：买家申请发票（增值税普票/专票），运营或自动开票对接（见开放问题 Q-12 开票渠道）；记录发票抬头、税号、状态。
- **REQ-A-054 (P1)** 账单中心：买家可见消费账单、订单流水、发票记录。
- **REQ-A-055 (P1)** 退款对账：退款触发分账资金归还（docs/04），更新三方账目。

## 3.6 信用与评价（P1/P2）
- **REQ-A-060 (P1)** 供给方信用分：基于履约 SLA、投诉、争议结果计算，影响排序与曝光。
- **REQ-A-061 (P2)** 买家对订单评价；供给方对买家备注。

## 3.7 数据看板（运营）
- **REQ-A-070 (P0)** 交易看板：GMV、订单数、成交均价、热门型号、供给/需求漏斗、撮合成功率。（运营后台 docs/08 复用）

## 3.8 验收要点
1. 机房入驻→审核→进件→上架→下单→支付→分账→对账→发票 全链路可走通（至少 1 条真实/沙箱链路）。
2. 余量锁定与超时释放正确，无超卖。
3. 平台佣金按配置正确进入分账。
4. 风控可拦截/冻结疑似挖矿订单（见 docs/09）。

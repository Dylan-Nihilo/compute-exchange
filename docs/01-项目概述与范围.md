# 01 · 项目概述与范围

## 1.1 背景

公司采用**轻资产模式**经营算力相关业务：**不自建算力机房、不自购大规模 GPU 集群**，而是做算力撮合、渠道分销、设备居间融资、AI 模型部署打包服务。业务矩阵包含五大板块：

1. **算力现货撮合交易平台**（类交易所挂牌）：短/中期合规算力分时·包月租赁、实时报价、线上撮合、订单结算。
2. **开源大模型一站式部署 + AI Token 代销/包销**（Token 工厂）。
3. **合规算力中心整机/机柜包销、算力设备融资租赁居间**。
4. **算力机房配套**：服务器、机电设备销售、机房施工总包居间。
5. **算力余量调度**：整合全国合规机房闲置间歇算力统一对外售卖。

本项目第一版（v1）落地形态为 **Web 平台**。

## 1.2 v1 目标

| 目标 | 说明 | 衡量 |
|------|------|------|
| G1 上线在线算力撮合交易能力 | 供给端可挂牌、需求端可下单、平台可撮合并完成在线支付与分账 | 完成 1 笔真实算力订单的"挂牌→下单→支付→分账→对账"闭环 |
| G2 Token 工厂可对外售卖 | 独立 newapi 部署，主平台可下单充值、同步额度、提供 API Key | 用户可在平台购买 Token 套餐并成功调用模型 |
| G3 全角色自助多端 | 买家/机房/设备厂商/资方可自助注册、资质入驻、使用对应工作台 | 4 类角色均能完成注册→审核→进入工作台 |
| G4 居间与金融板块获客 | 设备/机电/施工居间可登记需求并进 CRM；融资租赁可展示+留资 | 线索可被运营在后台跟进 |
| G5 合规可控上线 | 通过"交易开关受合规状态控制"实现先开发后取证的灰度上线 | 合规未齐备时，展示可用、真实成交收款受控 |

## 1.3 范围边界（In / Out of Scope）

### ✅ In Scope（v1 必做）

- 平台门户官网（五大板块业务展示、定价说明、合规与资质展示位）。
- 统一账户体系 + 全角色自助注册登录 + 实名/企业 KYC + 资质入驻审核。
- 算力商品挂牌、检索选型、分时/包月下单、撮合成交、订单履约状态流转。
- 易宝支付对接：充值、合单支付、平台抽佣分账、退款、对账。
- AI Token 工厂：独立 newapi 部署对接、套餐售卖、额度同步、用量账单。
- 多端工作台：买家端、机房供给端、设备厂商端、资方端。
- 居间业务：设备销售/机电施工居间需求登记、撮合线索、CRM、佣金台账。
- 融资租赁：展示页 + 需求留资表单（邮件/联系方式），不做系统化资方撮合。
- 运营管理总后台：入驻审核、商品/订单/资金/内容/权限管理。
- 风控与合规：挖矿特征识别与异常订单冻结（规则版）、资质审核台账、审计日志、内容合规（公告/上架内容审核）。
- 电子合同、发票申请、用量/资金账单。
- 基础数据看板（运营 GMV/订单/用户/资金）。

### ❌ Out of Scope（v1 不做，列入后续阶段）

- **真实算力的底层调度/隔离/网关**（第三层技术中台）：v1 不做实际算力编排，算力开通通过对接机房/云的既有控制台或人工/半自动方式完成；平台只管交易与履约状态。详见开放问题 [Q-09]。
- **融资租赁系统化撮合**（资方在线对接、合同、放款）：v1 仅展示+留资。
- **自建大模型训练/微调平台**。
- **App / 小程序原生端**（v1 仅 Web，移动端做响应式适配）。
- **国际化多语言**（v1 仅中文，预留 i18n 结构）。
- **跨境结算/外币**。
- **复杂金融衍生**（算力期货、份额认购等——亦属合规红线，明确不做，见第11章）。

> ⚠️ 关于"算力余量调度（板块5）"：v1 将其作为**货源策略**对待（运营侧集采闲置算力后以普通算力商品挂牌零售），**不做**统一调度网关这一技术系统。

## 1.4 阶段路线（与业务输入对齐）

| 阶段 | 周期（绝对） | 重点 | 本项目对应 |
|------|------|------|-----------|
| 阶段1 轻资产起步 | 2026 H2 起 | 居间分销、线下撮合、资质审核台账 | 本 v1 覆盖其线上化（门户+CRM+多端） |
| 阶段2 线上挂牌交易 | 与 v1 合并推进 | ICP/EDI/等保、撮合平台、Token 中台、算法备案 | **本 v1 核心**（交易+支付+Token+合规） |
| 阶段3 全产业链闭环 | 18 个月内 | 机电施工总包居间、融资租赁牌照、自有 Token 品牌、接入全国一体化算力网 | v1 之后迭代 |

> 说明：产品负责人已确认 v1 直接做"完整在线挂牌撮合交易所"，因此阶段1与阶段2在本项目中合并实现，靠"合规状态开关"控制真实交易开闸时点。

## 1.5 术语表（Glossary）

| 术语 | 含义 |
|------|------|
| 撮合 / 现货撮合 | 买卖双方在平台挂单，系统按规则匹配并促成成交 |
| 挂牌 / 上架 | 机房供给方将一段算力（GPU 型号/时段/单价/余量）发布为可售商品 |
| 分时租赁 | 按小时/天等时间粒度租用算力 |
| 包月 | 按月整体租用某规格算力 |
| 余量算力 | 机房闲置/间歇的可对外出售算力 |
| 包销 | 平台批量锁定某机房算力，自主定价独家售卖 |
| 代销 | 平台采购大厂合规模型 Token 流量池后零售 |
| Token 工厂 | 部署开源大模型 + 封装 OpenAI 标准 API + 计量计费的中台（本项目独立 newapi 实例） |
| newapi | 开源 AI API 网关/中转计费系统（现有 kaizo.top 在用；本项目新部署独立实例） |
| 二清 | 二次清算：无支付牌照主体截留并再分配他人资金，属违规红线 |
| 合单支付 | 易宝能力：用户一笔支付拆分进入多个收款商户账户 |
| 分账 | 易宝能力：按规则把一笔交易款实时拆给多个收款方（含平台佣金） |
| 进件/入网 | 商户在支付机构注册成为（二级）商户的过程 |
| KYC | 实名/企业身份与资质核验 |
| EDI 证 | 在线数据处理与交易处理业务经营许可证（增值电信 B21） |
| ICP 证 | 互联网信息服务业务经营许可证（经营性） |
| 等保 | 网络安全等级保护（本项目目标三级） |
| 算法备案 | 网信办生成式 AI/算法推荐备案 |
| 幽灵算力 | 来源不明、无资质、可能涉挖矿的非合规算力 |

## 1.6 设计原则

1. **合规优先**：任何功能不得突破挖矿红线与支付二清红线；交易开闸受合规状态控制。
2. **轻资产**：平台不持有算力、不截留资金；技术上不做底层算力编排（v1）。
3. **可灰度**：展示与撮合线索可先上线，真实成交/收款由开关控制。
4. **复用与隔离**：Token 工厂用独立 newapi（与现有 kaizo.top 物理隔离），通过账户/订单对接，不重写计量。
5. **多端一账户**：一套账户体系承载多角色，按资质解锁不同工作台与权限。
6. **可审计**：资金、订单、算力使用、内容变更全留痕，满足监管溯源。

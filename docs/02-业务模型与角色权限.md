# 02 · 业务模型与角色权限

## 2.1 四层架构回顾（业务输入映射）

| 层 | 名称 | 本 v1 是否实现 | 说明 |
|----|------|----------------|------|
| 第一层 | 算力供给层（外部合作，零自有硬件） | 部分（线上化登记/入驻） | 货源仍靠线下签约；平台做资质台账与上架 |
| 第二层 | 平台核心系统（交易所 + Token 工厂中台 + 居间后台） | **是（v1 核心）** | 本文档主体 |
| 第三层 | 调度与算力隔离技术方案 | **否（v1 不做底层调度）** | v1 仅做交易/履约状态与风控规则；实际隔离依赖机房 |
| 第四层 | 商业化运营分层（盈利模型） | 是（计费/佣金/分账落地） | 见 2.5 |

## 2.2 盈利模型（决定系统要算哪些钱）

| 收入项 | 计费方式 | 系统支撑 |
|--------|----------|----------|
| 算力撮合佣金 | 成交额 × 平台费率（建议 3%–8%，可按品类/供给方配置） | 分账时抽佣（docs/04） |
| 间歇余量算力差价 | 集采价与零售价价差 | 平台作为"包销"供给方自营上架，差价即毛利 |
| AI Token 代销/包销毛利 | 采购价与零售价价差、阶梯定价 | Token 套餐定价（docs/05） |
| 设备/机电销售佣金 | 居间成交额 × 佣金率 | 居间台账（docs/07） |
| 设备融资租赁居间费 | 撮合成功服务费 | 留资+线下，台账登记 |
| 大模型部署技术服务费 | 一次性部署费 + 运维年费 | Token 工厂订单/服务工单（docs/05） |

## 2.3 角色定义（全角色自助多端）

| 角色 | 英文键 | 自助注册 | 需资质审核 | 核心诉求 | 工作台 |
|------|--------|----------|-----------|----------|--------|
| 访客 | guest | — | — | 浏览、比价、留资 | 门户官网 |
| 算力买家（个人/企业） | buyer | ✅ | 实名/企业 KYC | 买算力、买 Token、对账 | 买家工作台 |
| 机房供给方 | supplier | ✅ | **机房资质强审核** | 上架算力、接单、收款、结算 | 供给方工作台 |
| 设备厂商/经销商 | vendor | ✅ | 企业资质 | 发布设备、接居间订单、佣金 | 设备厂商工作台 |
| 资方（融资租赁/基金） | funder | ✅（轻） | 企业资质 | 接收融资需求线索 | 资方工作台（v1 轻量：线索查看） |
| 平台运营 | operator | ❌（后台开通） | — | 审核、撮合、风控、对账、内容 | 运营总后台 |
| 平台管理员 | admin | ❌ | — | 账号权限、系统配置、合规开关 | 运营总后台（高权限） |

> 说明：一个登录账户可在通过对应资质后**解锁多重身份**（如既是买家又是供给方）。权限按"已认证身份 + 角色"动态计算。

## 2.5 核心端到端流程

### 流程 A：算力撮合交易主流程（核心闭环）

```
机房供给方                     平台                          算力买家
   │ 注册→提交机房资质            │                               │
   │ ───────────────────────────►│ 运营审核资质(排幽灵算力)        │
   │                             │ ◄──────审核通过, 易宝进件成二级商户
   │ 上架算力(型号/时段/单价/余量)│                               │
   │ ───────────────────────────►│ 商品上线(运营/规则审核)         │
   │                             │                               │
   │                             │ ◄──────────────────检索/比价/选型
   │                             │ ◄──────────────────分时/包月下单
   │                             │ 撮合匹配→生成订单→锁定余量        │
   │                             │ ◄──────────────────在线支付(易宝合单)
   │                             │ 资金按分账规则: 机房应得 + 平台佣金 │
   │ ◄───────────────────────────│ 通知接单/开通算力               │
   │ 开通算力(对接机房/人工)       │ ───────────────────►订单履约中→交付凭证
   │                             │ 履约监管(风控:挖矿识别)          │
   │                             │ 到期/续费/退款                  │
   │                             │ 对账→电子合同→发票              │
```

### 流程 B：Token 工厂购买流程

```
买家 → 选 Token 套餐/模型 → 下单支付 → 平台回调成功
     → 调用独立 newapi Admin API 充值额度/创建令牌
     → 返回 API Key + base_url → 买家调用模型 → 用量账单实时回显
```

### 流程 C：居间撮合流程（设备/机电/施工）

```
需求方填写居间需求(设备清单/施工范围/预算)
  → 进 CRM 线索池 → 运营/对应厂商认领
  → 线下报价洽谈 → 成交登记 → 佣金台账 → 结算
```

### 流程 D：融资租赁留资流程（v1 轻量）

```
访客/买家在融资板块填写需求(设备金额/期限/主体)
  → 留资(邮箱/电话) → 进 CRM → 运营分配给资方/线下跟进
  ✦ v1 不做在线方案匹配/合同/放款
```

## 2.6 状态机（关键对象）

### 机房入驻资质
`草稿 → 待审核 → 审核中 → 已通过 / 已驳回(可重提) → 已冻结`

### 算力商品
`草稿 → 待审核 → 在售 → 已售罄/已下架 → 已停用`

### 算力订单
`待支付 → 已支付(待开通) → 开通中 → 履约中 → 已完成 / 已取消 / 退款中 / 已退款 / 已冻结(风控)`

### 分账
`待分账 → 分账中 → 分账成功 / 分账失败(重试) → 已对账`

### 居间线索
`新建 → 待分配 → 跟进中 → 已报价 → 已成交 / 已关闭`

# 12 · 版权与第三方组件

> 你特别要求确认版权问题。本章梳理 v1 涉及的开源/商标/模型/素材版权，给出合规处理建议。**最终以法务复核为准。**

## 12.1 newapi（Token 工厂核心依赖）

- **事实**：本项目新部署独立 newapi 实例。newapi（new-api）为开源项目，但**开源协议需逐项核实**——部分 AI 网关项目采用 Apache-2.0 / MIT，也有项目带"附加使用条款/商用限制/不得移除版权标识"。
- **风险**：① 协议是否允许商用与二次分发；② 是否要求保留版权/署名；③ 是否禁止移除原作者标识或品牌。
- **建议**：
  1. 移交架构阶段**锁定所用 newapi 仓库与版本**，留存其 LICENSE 原文。
  2. 商用前确认协议允许 SaaS 商用；若有 copyleft（GPL/AGPL）条款，评估是否触发开源义务（AGPL 对 SaaS 尤其敏感）。
  3. 不擅自移除版权声明；如需去除前端品牌标识，确认协议允许。
- **开放问题 [Q-17]**：确定 newapi 具体仓库/版本与许可证。

## 12.2 推理框架与开源模型

- **vLLM / TensorRT-LLM**：vLLM 通常 Apache-2.0（商用友好）；TensorRT-LLM 属 NVIDIA，受其许可与 GPU 驱动/CUDA 条款约束，**商用部署需核对 NVIDIA 许可**。
- **开源大模型权重授权**（关键）：不同模型协议差异大——
  - 部分"开源"模型对**商用有条件限制**或月活阈值（如某些 Llama 系列条款、Qwen/部分模型的商用许可）。
  - **必须逐一核对每个上架模型的权重许可证是否允许商业 API 售卖**。
- **建议**：建立"模型版权台账"，每个上架模型登记：权重许可证、是否允许商用 API、是否需署名、是否有地域/用途限制、备案状态。
- **开放问题 [Q-18]**：v1 首批上架哪些模型？逐一过授权。

## 12.3 代销上游 Token（大厂模型流量池）

- 代销大厂模型 Token 须有**合法分销/代理授权**，遵守上游服务条款（转售许可、限速、内容政策、品牌使用）。
- 若上游为境外（OpenAI/Anthropic 等）：注意境内合规可用性、数据出境、是否允许转售（多数官方条款**限制再分发/转售**）——需评估。
- **建议**：仅接入允许转售或有正式渠道授权的上游；留存授权文件。
- **开放问题 [Q-19]**：代销上游清单与授权确认。

## 12.4 支付与第三方 SDK

- 易宝 SDK/接口：遵守易宝开放平台协议与品牌使用规范；密钥保密。
- 短信/实名/电子签/发票等第三方服务：均为商用授权采购，遵守各自条款（见 NFR/开放问题）。

## 12.5 平台软件栈开源协议

- v1 将使用大量前后端开源框架/库（React/Vue、Node/Java/Go、UI 组件库、图表库等）。
- **建议**：建立 SBOM（软件物料清单），统一审查协议：
  - ✅ 优先 MIT/Apache-2.0/BSD（宽松，商用友好）。
  - ⚠️ 谨慎 GPL/AGPL/LGPL（copyleft，可能要求开源/动态链接限制），SaaS 场景尤其注意 AGPL。
  - ⚠️ 商用受限协议（部分 UI 库/图表库企业版、字体）需购买商用授权。
- **开放问题 [Q-20]**：技术选型在架构阶段产出 SBOM 并过一遍许可证。

## 12.6 视觉素材 / 字体 / 图标 / 商标

- **字体**：商用须用可商用字体（如思源/部分免费商用字体），**禁止使用未授权商用字体**（微软雅黑等需授权）。
- **图标/插画**：用可商用授权素材（自绘、付费授权、CC0），留存授权凭证。
- **品牌/商标**：决策 D6 品牌暂定。命名后须做**商标检索与注册**，避免侵权；域名与商标一致性。
- **生成式 AI 生成的素材**：若用 AI 生成 logo/配图，注意生成内容版权与平台条款（商用授权范围）。
- **参考既往经验**：项目记忆中"即梦 AI 提示词规则"等素材生产规范可复用于本项目运营物料。
- **开放问题 [Q-21]**：品牌命名 + 商标注册计划。

## 12.7 内容与数据版权

- 门户案例、合作伙伴 logo 使用须获授权。
- 训练/语料（若自部署模型涉及）：语料来源合法性是算法备案安全评估的硬指标（第11章）。

## 12.8 版权合规清单（移交架构/实施阶段执行）
1. newapi 仓库版本 + LICENSE 锁定与商用确认（Q-17）。
2. 上架模型权重许可证逐一台账（Q-18）。
3. 代销上游转售授权确认（Q-19）。
4. 全栈 SBOM + 许可证审查（Q-20）。
5. 字体/图标/素材商用授权留存。
6. 品牌命名 + 商标检索注册（Q-21）。
7. 第三方 SDK/服务协议遵从。

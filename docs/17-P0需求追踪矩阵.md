## 17.1 使用规则

- 每个 P0 REQ ID 只出现一行；需求增删或优先级变化必须同步更新本表并重新核对唯一计数。
- 未显式标注 `(P0)` 的 `REQ-H-010` 至 `REQ-H-015` 不纳入 P0 计数；在 17.3 单列是为了避免链上事件实现遗漏。
- “页面/用户动作”可以是前端路径，也可以是无直接页面的服务端或运维动作。
- “守卫”覆盖角色权限、认证、合规、资金与安全边界；后端守卫不能由前端显示状态替代。
- “契约/状态”指当前领域契约或计划中的生产边界；第三方能力均以沙箱和书面确认结果为准。
- A 阶段使用 Mock 验证用户闭环；B 阶段才代表真实 API、资金、KYC、newapi、存证与合规生产能力。

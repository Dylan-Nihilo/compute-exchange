# 08 · 功能需求 · 运营管理总后台

> 需求编号：`REQ-F-xxx`。运营(operator)与管理员(admin)使用。

## 8.1 审核中心

- **REQ-F-001 (P0)** 机房入驻资质审核：队列、详情、通过/驳回/补件、审核留痕，触发易宝进件。
- **REQ-F-002 (P0)** 设备厂商/资方资质审核。
- **REQ-F-003 (P0)** 商品上架审核：算力商品、Token 模型/套餐的合规与价格审核。
- **REQ-F-004 (P0)** 内容审核：门户内容、公告、案例发布前审核。

## 8.2 商品与供给管理

- **REQ-F-010 (P0)** 算力商品管理：全平台商品查看、强制下架、改价干预、平台自营商品上架（集采/包销）。
- **REQ-F-011 (P0)** Token 模型/套餐管理：上下架、定价、倍率、备案号维护。
- **REQ-F-012 (P1)** 货源/供给池台账：合作机房列表、资质、可供算力概览（对应"合规算力货源池"）。

## 8.3 订单与交易管理

- **REQ-F-020 (P0)** 全平台订单查看/检索（算力、Token、居间）。
- **REQ-F-021 (P0)** 订单干预：协助开通、改状态、关闭、退款发起、争议处理。
- **REQ-F-022 (P0)** 撮合干预：异常订单处理、人工撮合大额询价单转订单。

## 8.4 资金与财务管理

- **REQ-F-030 (P0)** 分账规则配置：撮合费率（按品类/供给方/活动）、分账方、佣金规则。
- **REQ-F-031 (P0)** 对账中心：与易宝对账、差异处理、三方账目台账。
- **REQ-F-032 (P0)** 结算管理：供给方结算单、佣金报表、退款与资金归还审批。
- **REQ-F-033 (P1)** 财务报表导出与看板（GMV、佣金收入、Token 毛利、居间佣金）。

## 8.5 CRM 与线索管理

- **REQ-F-040 (P0)** 线索池：设备居间、施工居间、融资租赁留资统一管理。
- **REQ-F-041 (P0)** 线索分配/认领/跟进/转化登记。
- **REQ-F-042 (P1)** 客户管理：企业/个人客户档案、跟进记录、标签。

## 8.6 内容管理（CMS）

- **REQ-F-050 (P0)** 门户页面/板块介绍/案例/合作伙伴/合规资质展示位编辑。
- **REQ-F-051 (P0)** 公告管理：发布/编辑/上下线（注意 HTML 渲染规范，参考既有 newapi 公告 markdown 陷阱经验：HTML 公告需单行 minify）。
- **REQ-F-052 (P1)** Banner/活动位、SEO 元信息维护。

## 8.7 用户与权限管理

- **REQ-F-060 (P0)** 用户管理：检索、详情、身份、认证状态、冻结/解冻。
- **REQ-F-061 (P0)** RBAC 权限：运营角色与权限点配置（审核、财务、风控、内容等子角色分权）。
- **REQ-F-062 (P0)** 操作审计日志：后台所有敏感操作留痕（操作人、对象、前后值、时间、IP），可检索导出。

## 8.8 系统配置（admin）

- **REQ-F-070 (P0)** **合规状态开关**：控制真实交易与收款是否开闸（依据 ICP/EDI/等保/支付进件状态），支持三态：仅展示 / 限额真实交易（允许支付，受单笔和累计限额）/ 全量真实交易。详见第13章上线策略。
- **REQ-F-071 (P0)** 基础配置：费率默认值、通知模板、支付参数、newapi 对接参数（密钥走密管，不明文）。
- **REQ-F-072 (P1)** 字典/枚举管理（GPU 型号、地域、模型等）。
- **REQ-F-073 (P1)** 数据备份与导出策略配置（参考既有 MySQL 自动备份实践）。

## 8.9 数据看板

- **REQ-F-080 (P0)** 运营总览：用户/订单/GMV/佣金/Token 毛利/线索转化/供给规模。
- **REQ-F-081 (P1)** 风控看板：异常订单、冻结、挖矿告警统计（docs/09）。

## 8.10 验收要点
1. 审核→进件→上架→交易→分账→对账→结算全链路在后台可管可查。
2. 合规开关能正确控制真实交易开闸与灰度。
3. RBAC 分权与操作审计完整。

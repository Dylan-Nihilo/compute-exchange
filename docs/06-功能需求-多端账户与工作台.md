# 06 · 功能需求 · 多端账户与工作台

> 需求编号：`REQ-D-xxx`。决策 D5：**全角色自助多端**。一套账户体系，按已认证身份解锁多工作台。

## 6.1 统一账户体系

- **REQ-D-001 (P0)** 统一账户：手机号/邮箱注册登录，一个账户可承载多重身份（buyer/supplier/vendor/funder），身份按资质认证解锁。
- **REQ-D-002 (P0)** 登录方式：手机号+短信验证码、账号密码；图形/滑块验证码防刷；登录失败按 IP/账号限流（参考既有 newapi 登录限流经验，防撞库）。
- **REQ-D-003 (P0)** 注册即默认 buyer 身份（最小权限）；申请其它身份须提交对应资质审核。
- **REQ-D-004 (P1)** 第三方登录（微信扫码/企业微信）——P1，便于企业用户。
- **REQ-D-005 (P0)** 账户安全：密码加密存储、登录设备/IP 记录、敏感操作二次验证、会话管理与登出。
- **REQ-D-006 (P1)** 子账户/成员：企业账户下可加子成员并分配角色（供给方/买家企业场景）。

## 6.2 实名与企业认证（KYC）

- **REQ-D-010 (P0)** 个人实名：姓名+身份证（三要素/人脸核验，渠道见 Q-13）。
- **REQ-D-011 (P0)** 企业认证：企业名称、统一社会信用代码、营业执照上传、法人信息、对公账户验证。
- **REQ-D-012 (P0)** 认证状态机：未认证 / 认证中 / 已认证 / 已驳回；下单、上架、进入易宝结算入口等关键动作受认证门槛控制。
- **REQ-D-013 (P0)** 资质与支付进件打通：企业认证信息复用于易宝二级商户进件（docs/04），避免重复填报。

## 6.3 买家工作台（buyer）

- **REQ-D-020 (P0)** 我的算力订单：列表/详情/状态、交付凭证、续费、退款、工单。
- **REQ-D-021 (P0)** 我的 Token：已购服务额度、API Key、用量账单、购买套餐。额度不是现金余额。
- **REQ-D-022 (P0)** 账单中心：消费流水、发票申请与记录。
- **REQ-D-023 (P1)** 我的收藏/比价、求购单（若开 REQ-A-033）。
- **REQ-D-024 (P1)** 消息中心：订单/履约/到期/活动通知。

## 6.4 机房供给方工作台（supplier）

- **REQ-D-030 (P0)** 机房资质管理：资质上传、审核状态、到期预警。
- **REQ-D-031 (P0)** 算力商品管理：上架/编辑/上下架、余量与时段、调价规则。
- **REQ-D-032 (P0)** 订单管理：接单、标记开通、回填交付凭证、履约状态、到期处理。
- **REQ-D-033 (P0)** 结算中心：结算单、分账流水、待结/已结、对账。
- **REQ-D-034 (P1)** 数据看板：在售/成交/收入/占用率。
- **REQ-D-035 (P1)** 售后/工单处理。

## 6.5 设备厂商工作台（vendor）

- **REQ-D-040 (P0)** 企业资质管理。
- **REQ-D-041 (P0)** 设备/服务发布：服务器、机电、UPS、制冷等设备信息（型号、规格、价格/可议价、库存、地域）。
- **REQ-D-042 (P0)** 居间订单/线索：接收并认领居间需求、报价、成交登记。
- **REQ-D-043 (P0)** 佣金台账：居间成交佣金、结算状态。
- **REQ-D-044 (P1)** 数据看板：线索量、成交额、佣金。

## 6.6 资方工作台（funder，v1 轻量）

- **REQ-D-050 (P1)** 企业资质管理。
- **REQ-D-051 (P1)** 融资需求线索查看：运营分配的融资租赁需求线索（设备金额、期限、主体、联系方式）。
- **REQ-D-052 (P1)** 线索处理状态：认领/跟进/反馈（线下成交，系统仅登记）。
- **REQ-D-053 (P2)** 在线方案提报（后置，v1 不做撮合）。

## 6.7 通用能力（各端共用）

- **REQ-D-060 (P0)** 个人/企业信息维护、联系方式、收票信息。
- **REQ-D-061 (P0)** 消息/通知中心（站内信，短信/邮件可配）。
- **REQ-D-062 (P1)** 操作日志（用户侧可见自身关键操作）。
- **REQ-D-063 (P0)** 工作台首页：按身份聚合待办（待支付、待接单、待审核补件、待结算等）。

## 6.8 验收要点
1. 同一账户认证 supplier 后，工作台出现供给方菜单且权限正确。
2. 企业认证信息能复用到易宝进件。
3. 各端待办聚合准确，关键动作受认证/审核门槛控制。
